# 第一步：确认环境
打开你的终端，依次运行这两条命令，把结果告诉我：


python3 --version

pip3 --version
为什么要确认这两个？

python3 --version：确认 Python 可以正常调用，版本是你说的 3.14.3
pip3 --version：pip 是 Python 的包管理器（相当于 App Store），后面要用它来安装 Gemini 的库。如果 pip 没有或版本很旧，安装库会出问题

---

# 创建虚拟环境（venv）
在安装任何库之前，先做这一步：


python3 -m venv .venv
然后激活它：


source .venv/bin/activate
激活后，你的终端提示符会变成 (.venv) ...，表示"进入"了虚拟环境。

为什么要做这个？

直接用系统 Python 装库，会把所有项目的依赖混在一起——项目 A 需要库的 1.0 版、项目 B 需要 2.0 版，就会互相打架。

虚拟环境的作用是：给这个项目单独开一个隔离的空间，你在里面装的库只属于这个项目，不影响系统也不影响其他项目。

这是 Python 项目的标准做法，每个项目一个 venv。

做完之后终端应该长这样：


(.venv) a11111@... pet-triage %

---
创建（python3 -m venv .venv）：每个项目只做一次。
它会在你的项目目录下生成一个 .venv 文件夹，里面存放这个项目专属的 Python 和所有库。文件夹建好就一直在那，不用重复建。

激活（source .venv/bin/activate）：每次开新终端都要运行一次。
"激活"只是告诉当前这个终端会话"去用 .venv 里的 Python"。关掉终端、换一个终端窗口，这个设置就消失了，下次要重新激活。

---
第三步：安装 Gemini 库
在激活了 .venv 的终端里运行：

`pip install google-genai`
为什么是 google-genai 而不是别的名字？

Google 的 Gemini API 有一个官方 Python 库，它的包名就叫 google-genai。这条命令会把这个库下载到你的 .venv 里（不是系统里）。

pip 会打印一堆 Downloading... Installing...，最后看到 Successfully installed 就成功了。

装好之后，验证一下装进去了：

`pip list | grep google`
这条命令是什么意思？

pip list：列出这个环境里装了哪些库
|（管道符）：把左边命令的输出，当成右边命令的输入
grep google：只显示含"google"字样的行，过滤掉无关的

# 导入 Python 内置的 os 模块。os 负责和操作系统打交道，这里用它来读取环境变量。
import os
# python-dotenv 这个库专门负责读取 .env 文件，把里面的内容变成环境变量
from dotenv import load_dotenv

### Q:import os 和 from dotenv import load_dotenv 不是重复吗？

不一样，它们做的是两件不同的事，要配合才能工作：

load_dotenv() 的作用：把 .env 文件里的内容读出来，写进系统环境变量。它像一个"搬运工"，把文件里的 GEMINI_API_KEY=xxx 搬到内存里。
os.getenv("GEMINI_API_KEY") 的作用：从系统环境变量里读取值。它像一个"取货员"，去内存里取那个值。
顺序是：.env 文件 → load_dotenv() 搬运 → 系统环境变量 → os.getenv() 取出来。

### Q 为什么所有东西放一个文件就能跑？

Python 执行 .py 文件的方式非常简单：从第 1 行开始，逐行往下读，遇到什么就执行什么。就像照着菜谱做饭，所有材料和步骤写在一张纸上，从头做到尾。

不需要什么特别的"入口"或配置，文件本身就是程序。
